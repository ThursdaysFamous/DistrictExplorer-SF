First engine release — the fenced engine becomes a published, hash-verified artifact (docs/MECHANIZATION_PLAYBOOK.md, Conversion 1).

- 45 ENGINE blocks from index.html + 2 from sw.js, concatenated in document order into `engine.bundle.js`; `engine.manifest.json` carries the version, the ordered block list with per-block source file and byte length, and the bundle sha256.
- No engine code changed in this release: the bundle is byte-identical to the fenced blocks already deployed. It exists so forks can pin it in `engine.lock.json` and assemble from it at deploy time instead of hand-porting diffs.
- Consume: `gh release download` the bundle + manifest, `sha256sum --check` against the lockfile pin, then `python3 scripts/apply_engine.py` followed by the merge gate and the smoke test. `apply_engine.py` (and the fence parser it imports, `check_engine_parity.py`) ship as assets on this release — the release is the distribution channel for shared scripts.
