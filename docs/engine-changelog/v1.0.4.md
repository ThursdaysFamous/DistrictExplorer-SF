Location-relevance hiding (`mod.coverage`) тАФ the **first engine-code change since v1.0.0** (v1.0.1тАУv1.0.3 shipped byte-identical bundles; this one changes engine bytes, sha `e2be4fdbтАж46dcc1`). Strictly additive: a layer that declares no `coverage` takes exactly the pre-v1.0.4 code path, so a fork that declares nothing (NYC today) is behaviorally unchanged.

- **New optional layer-contract field `coverage(point) => boolean | Promise<boolean>`** тАФ declares where a layer's data applies. Where the selected point falls outside a layer's coverage the layer *hides*: toggle block, result card, map overlay, highlight, POI pin, hover-snapshot row, and relationship outlines are all suppressed, and its query (with any boundary fetch the query would trigger) is skipped. `state.layersOn` is never mutated тАФ `layers=` permalinks survive an out-of-coverage detour and the layer reappears the moment a selected point re-enters coverage. A throwing coverage test fails **open** (the layer runs as if undeclared). Group sections whose top-level layers are all out of coverage hide as a unit.
- Engine surface (in `layer-registry` / `overlay-cards`): `relevant`/`relevanceSeq`/`relevancePromise` runtime fields; `layerRelevance()` (per-selection memoized, fail-open); `refreshLayerBlockHidden()` (single writer of `block.hidden`, composing relevance with the `subOf` parent-off rule, releasing keyboard focus before hiding); `refreshGroupVisibility()`; `setLayerRelevant()` (suppress/reattach incl. fill-opacity rescale + relationship-outline refresh); `updateAllLayerRelevance()` from `runAllActiveLayerQueries()`; `runLayerQuery` split into a coverage gate + `runLayerQueryAt` тАФ cards are invalidated (`setCardLoading`) *before* the coverage test so a slow/failing coverage source can never leave a previous point's results attributed to a new selection.
- Relevance composes across surfaces: `hoverContainingLayers`, `drawRelationshipOutlines`, and `relationshipParentFeature` skip `relevant === false` layers (a relevance-hidden pinned parent contributes nothing but stays pinned, resurrecting on return); `activeLayerCount()` counts only visible actives so fill-opacity scaling ignores hidden layers.
- Factories `registerPolygonLayer` / `registerNearestPointLayer` / `registerSchoolZone` / `registerCpsNetwork` pass `opts.coverage` through.
- Consume-side scripts (`apply_engine.py`, `check_engine_parity.py`, `generate_metro_files.py`, `metro-worksheet.schema.json`) are unchanged.
- Fork-side reference (travels in the CHI repo, not the assets): CHI declares `chicagoCoverage` (ERSB tiling тИк community areas тАФ coverage tests **over-approximate** so a sliver miss yields an honest empty card, never a false hide) on its 16 city-scoped layers and `cookCountyCoverage` (CCBR tiling тИк IL-Supreme District 1) on its 2 Cook-scoped layers; its smoke test's negative-point checks assert hide + `state.layersOn` stability.
